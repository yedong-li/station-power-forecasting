# 融合场站异构性与运维事件感知的 Transformer 光伏电站短期功率预测方法：论文框架

## 1. 总体论证主线

本文不宜直接写成“提出一个全新 Transformer 模型”，而应写成“面向真实电站运维预测场景，对场站异构性、运维事件缺失和模型结构适配问题进行系统建模与验证”。建议全文按照以下逻辑展开：

**现实问题提出 → 现有方法不足 → 针对性模型设计 → 实验验证价值 → 边界与应用场景说明。**

核心论证链为：

1. 中国不同地区光伏电站自然条件和设备条件差异大，单一通用模型难以兼顾所有场站。
2. 真实电站存在计划检修、故障排查、限电、设备停机等运维事件，若不纳入模型，会造成异常时段预测偏差和检修时段预测残留。
3. 传统统计模型、循环网络和常规 Transformer 多以气象与历史功率为主要输入，对场站特异性和稀疏运维事件的刻画不足。
4. 因此，本文设计“场站专属建模 + 运维事件嵌入 + O&M Gate 门控抑制”的预测流程，并通过分组实验验证每个设计环节的贡献。

## 2. 需要突出的三个问题

### 2.1 场站空间异构性问题

中国不同地区光伏电站所处自然环境差异显著。东北、西北、华东、西南等区域在辐照度、温度、湿度、风沙、降雨和积灰速度方面存在明显差异；同时，不同场站的光伏板材料、组件老化程度、逆变器配置、装机容量和转换效率也不一致。

这些差异会导致同样的气象输入在不同场站下对应不同的功率响应。如果使用一个全局模型同时拟合多个场站，模型容易学习到“平均化”的响应关系，进而在具体场站上产生负迁移。因此，本文需要把“地区特性/场站特性”作为第一个核心问题，并用场站专属建模实验进行验证。

### 2.2 运维事件缺失问题

真实电站并非始终处于正常发电状态。常规检修、设备故障排查、逆变器停机、线路检修、限电调度等事件会直接改变实际出力。如果模型只使用辐照度、温度和历史功率，可能在检修或停机时仍预测出较高功率，形成预测残留。

因此，本文需要把运维事件从“附属日志”提升为预测模型的显式输入，将计划检修、故障和限电等事件编码进模型，并说明哪些未来事件在预测时可获得，哪些未来事件必须屏蔽以避免信息泄露。

### 2.3 现有模型适配不足问题

传统统计模型和机器学习模型对复杂非线性时序的表达能力有限；LSTM/GRU 等循环模型对长时依赖和突发状态切换的响应存在局限；常规 Transformer 虽能建模长时序依赖，但若输入中没有运维事件，仍无法解释由检修、故障和限电造成的非气象功率变化。

因此，本文不应只强调“用了 Transformer”，而应强调对 Transformer 输入结构和预测头的任务适配：一方面引入事件嵌入，另一方面在计划检修等确定性事件下引入 O&M Gate 门控抑制，减少预测残留。

## 3. 模型设计对应关系

| 现实问题 | 对应设计 | 论文中应强调的作用 |
| :--- | :--- | :--- |
| 地区自然条件、积灰速度、组件材料和效率差异大 | 场站专属建模 | 降低多场站混合训练带来的负迁移，使模型更贴合本地功率响应 |
| 检修、故障、限电等运维事件未进入预测模型 | 运维事件嵌入 | 将离散事件转化为可学习向量，与气象和功率时序融合 |
| 检修停机时模型仍可能预测残留功率 | O&M Gate 门控分支 | 在计划检修已知时对输出施加可微抑制，降低检修片段残留 |
| 常规模型对长时依赖和非平稳状态切换刻画不足 | Transformer 编解码结构 | 捕获历史功率、气象和事件序列之间的长时依赖关系 |

## 4. 建议论文结构

### 摘要

摘要按“四句话”组织：

1. 说明真实电站功率预测面临空间异构、运维事件和检修残留问题。
2. 提出场站专属建模、运维事件嵌入和 O&M Gate 的整体方法。
3. 给出核心实验结果，例如场站专属建模 MAE 降幅、运维事件融合降幅和检修时段 MAE。
4. 明确贡献边界：本文重点是面向电站运维场景的建模流程和可复现实验，而不是宣称全新 Transformer 理论。

### 0 引言

引言建议按以下顺序展开：

1. 新能源并网和短期功率预测的重要性。
2. 真实电站应用与标准预测任务之间的差异。
3. 提出三个具体痛点：
   - 地区与场站差异大，单一模型难以兼顾；
   - 运维事件未纳入模型，导致异常时段误差放大；
   - 现有模型结构未针对检修停机和事件可获得性进行适配。
4. 给出本文方法概述。
5. 列出本文贡献：
   - 场站专属建模策略；
   - 运维事件嵌入融合；
   - O&M Gate 门控输出抑制；
   - GitHub Actions 可复现实验协议。

### 1 相关工作

建议分为三类综述：

1. 光伏短期功率预测方法：ARMA、SVR、LSTM/GRU、TCN、Transformer、Informer、Autoformer、PatchTST。
2. 多源信息融合预测：NWP、辐照度、温度、历史功率、设备状态、运维日志。
3. 物理先验、规则后处理与约束预测：说明现有方法如何处理非负功率、限电、停机和后处理规则，并引出本文的可微门控设计。

### 2 问题定义与数据建模

本章应重点把“为什么不能直接用普通模型”形式化。

建议包含：

1. 预测任务定义：历史窗口、预测步长、输入特征和输出目标。
2. 场站异构性定义：地区、气象、装机容量、组件效率、积灰速度、设备老化。
3. 运维事件字典：正常、计划检修、设备故障、电网限电。
4. 未来事件可获得性约束：
   - 计划检修通常可提前获得；
   - 突发故障和临时限电在预测时不可提前知道，应屏蔽为正常状态。
5. 数据切分与标准化原则：按时间切分训练/测试，避免数据泄露。

### 3 模型设计

模型设计章节建议按“对应问题逐一解决”的方式写，而不是单纯堆叠网络模块。

1. 场站专属建模：
   - 每个场站独立训练模型；
   - 解决地区、设备和容量差异造成的负迁移。
2. 运维事件嵌入：
   - 将离散事件编码为连续向量；
   - 与气象、历史功率和未来气象输入融合。
3. Transformer 编解码结构：
   - 编码历史功率、气象和事件序列；
   - 解码未来气象和可获得的计划事件。
4. O&M Gate 门控分支：
   - 生成原始预测功率；
   - 根据事件偏置生成输出抑制比例；
   - 在计划检修时降低预测残留。
5. 训练目标和推理流程：
   - 损失函数；
   - 标准化/反标准化；
   - 非负功率处理；
   - 未来故障与限电屏蔽策略。

### 4 实验设计

实验章节要直接证明本文价值，建议分为五组实验。

#### 实验一：场站专属建模 vs. 空间共享模型

目的：证明“中国地区差异、设备差异、组件效率差异导致一个模型难以兼顾”。

对比：

| 模型 | 目的 |
| :--- | :--- |
| 空间共享多站模型 | 模拟一个模型同时拟合多个地区场站 |
| 场站专属模型 | 验证本地化建模是否降低负迁移 |

应报告各场站 MAE、RMSE 和提升率。

#### 实验二：无运维事件 vs. 融合运维事件

目的：证明检修、故障、限电等运维事件必须进入模型。

对比：

| 模型 | 输入 |
| :--- | :--- |
| 基线 Transformer | 气象 + 历史功率 |
| O&M Aware Transformer | 气象 + 历史功率 + 运维事件 |

应分别报告总体时段、正常时段和运维事件时段误差。

#### 实验三：门控消融实验

目的：证明模型结构调整的价值，而不是只靠事件拼接。

对比：

| 模型 | 说明 |
| :--- | :--- |
| 不含事件 Transformer | 无法感知检修状态 |
| 事件嵌入但无门控 | 能利用事件，但仍可能有残留 |
| 事件嵌入 + O&M Gate | 端到端降低检修残留 |
| 规则后处理基线 | 工程上限参考，不参与端到端训练 |

重点分析计划检修时段 MAE。

#### 实验四：门控偏置敏感性实验

目的：证明 O&M Gate 的参数设置不是随意给出的。

设置不同 $\beta_1$，观察计划检修 MAE 变化，说明偏置过小残留较大，偏置过大可能进入饱和区，因此选择折中参数。

#### 实验五：预测跨度敏感性实验

目的：说明模型在不同预测步长下的表现边界。

设置 12 h、24 h、48 h、72 h 等预测窗口，报告 MAE 和 RMSE。若结果不单调，不应强行解释为长跨度更优，而应说明样本构成和事件覆盖比例可能影响结果。

### 5 结果讨论

结果讨论建议围绕三个问题回答：

1. 场站差异是否真的影响模型？
   - 用实验一回答。
2. 运维事件是否真的提供额外信息？
   - 用实验二回答。
3. 模型结构调整是否有必要？
   - 用实验三和实验四回答。

同时要说明边界：

1. 当前公开数据为样例/脱敏构造数据，真实生产数据需要进一步验证。
2. 外部模型如 LSTM、Informer、PatchTST 尚需在统一协议下补充公平对比。
3. O&M Gate 是可微输出抑制机制，不等同于严格物理约束求解器。

### 6 结论与展望

结论应回到三个原始问题：

1. 针对地区和设备差异，场站专属建模降低了空间共享模型的误差。
2. 针对运维事件缺失，事件嵌入显著改善了场站 C 的总体和事件时段误差。
3. 针对检修预测残留，O&M Gate 在端到端框架内进一步降低计划检修时段误差。

展望建议写：

1. 引入真实长周期、多区域 SCADA 与工单数据。
2. 补充更多公平外部基线。
3. 融合非结构化运维文本日志。
4. 研究积灰、组件老化、清洗周期等慢变量对功率衰减的建模。
